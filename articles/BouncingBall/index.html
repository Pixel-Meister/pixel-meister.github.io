Bouncing Ball

To kick things off, I'd like to start with a bouncing ball. I'm just journaling my thought process for solving this problem.

I'm using p5.js to handle animation; however, I think as long as concepts are there you can apply this to whatever language and library you choose.

First, let's set up the scene. We draw an ellipse on the screen.

In p5, there are two main functions. There's setup, which is code that runs continuosly. You can specify otherwise, like having a loop or having draw only run once.

Looking at the p5.js reference for 2D objects we can see that the ellipse command sounds perfect. It has x and y coordinates and height and width arguments.

Sweet, so in our setup function we can set our canvas to the default 400x400 and we'll use the ellipse function in the draw funtion to create our circle.

Next, we'll try moving it. Since draw runs multiple times, we can change the X or Y positions each time draw is run. In this case, we're just adding 1 to the ellipse's x and y coordinates each time draw is called.

Cool! So we have understand that we can draw an ellipse and adjust its position. To have the ball bounce though, we need to change the direction if it reaches the edge of the screen.

Let's check if our ball position has reached either the furthest edge or the closest edge. We can check our canvas's size and if we're larger than the that or smaller than zero we reverse our direction. Let's start with just the X axis.

Hey, a bouncing ball, big ol pat on the back for ourselves! We extend that same logic to the y axis, and fingers crossed it works...
